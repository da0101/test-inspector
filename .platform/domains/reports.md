# reports

## What this domain does

Produce shareable artifacts from the in-memory state of a scan — primarily Markdown today, with JSON and HTML deferred. The Markdown report is what a developer pastes into a PR description or hands to a reviewer; the investigation export is what they paste into a code-review comment.

## Backend / source of truth

- **Top-level report renderer:** `src/services/report.ts` — builds the "Test Inspector Report" Markdown: summary KPIs, changed-file risk, quality findings, recommended commands.
- **Investigation export:** `src/services/investigationReport.ts` — renders the per-file or per-feature drilldown as Markdown for `exportInvestigation` / `exportFeatureInvestigation`.
- **Test fixtures:** `test/fixtures/test-inspector-report.md` is the golden master compared against in `test/unit/report.test.ts` — any deliberate format change requires updating it.

## Frontend / clients

- Commands: `exportReport`, `exportInvestigation`, `exportFeatureInvestigation`.
- Saved files land in the workspace (path chosen by the user via `vscode.window.showSaveDialog`).

## API contract locked

- The Markdown format is the contract — both the top-level report and the investigation export. `test/unit/report.test.ts` enforces structure; updates need fixture updates in the same PR.
- Every section is independently readable — a user who only cares about "Changed Files Risk" doesn't need to read the rest.
- No secrets in any export. The renderer redacts API keys defensively, even though they don't enter the render path today.
- File paths are workspace-relative wherever possible (cleaner diffs, no leaking of absolute paths).
- Reports include a "generated by Test Inspector vX.Y.Z" footer with the version pulled from `package.json` — so a stale report is identifiable.

## Key files

## Decisions locked

- Markdown is the primary format (D — distribution channel is PRs and reviews). JSON export (for automation) and HTML export are Phase 10 deferred.
- Reports never include LLM enrichment unless the user invoked the LLM path explicitly; the LLM block is segregated so the deterministic content is identifiable.
- A PR-readiness report (with explicit blockers / nice-to-haves) is a Phase 10 deliverable distinct from the generic top-level report.
- Reports are local files only — no upload, no clipboard mutation, no telemetry. Save dialog is the only egress.
- Snapshot-store under `.test-inspector/` (Phase 11) is where rolling reports will live; today there is no on-disk snapshot.
