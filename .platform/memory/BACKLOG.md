# Backlog

Deferred issues and out-of-scope work discovered during active development.

**Loading rule:** Do NOT load at session start. Load only when:
- The user explicitly asks to review or prioritize the backlog
- An agent is appending a new entry it found during active work

**Append rule (for agents):** When you encounter a real limitation, tech debt item, or missing feature that is out of scope for the current task — do NOT fix it, do NOT create a work stream for it. Just append a row here and move on. Priority guide: `high` = causes user-visible bugs or data loss at scale / `medium` = degrades UX or requires workaround / `low` = nice-to-have or edge case.
